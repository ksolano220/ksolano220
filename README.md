# Katherine Solano

Data Analyst focused on healthcare systems and operational decision-making.

I build data projects that analyze real public health datasets — identifying access gaps, cost patterns, and resource allocation opportunities. I also work on AI safety, specifically runtime control for autonomous agent systems.

---

## Healthcare Analytics

- [**Healthcare Access Risk Analysis**](https://github.com/ksolano220/healthcare-access-risk-analysis) — Scored 3,200+ US counties by healthcare access risk using uninsured rates and hospital density. Interactive choropleth map.
- [**Medicare Billing Analysis**](https://github.com/ksolano220/medicare-billing-analysis) — SQL analysis of 145k CMS Medicare records across 3,000+ hospitals. Cost outliers, markup patterns, rural vs urban comparisons.
- [**ED Wait Time Dashboard**](https://github.com/ksolano220/ed-wait-time-dashboard) — Streamlit dashboard analyzing emergency department wait times across 4,000+ hospitals with state-level comparisons and volume breakdowns.
- [**Clinical Trial Trends**](https://github.com/ksolano220/clinical-trial-trends) — Exploratory analysis of 10,000 trials from ClinicalTrials.gov. Sponsor patterns, phase completion rates, geographic distribution.

## AI Safety

- [**Sentra**](https://github.com/ksolano220/sentra) — Runtime execution control layer for autonomous AI agents. Deterministic policy enforcement, cumulative risk tracking, 3-strike shutdown. Includes a Python SDK for drop-in integration.
- [**Autonomous Claims Workflow**](https://github.com/ksolano220/autonomous-claims-workflow) — Multi-agent system powered by IBM watsonx.ai and Granite. Demonstrates how Sentra prevents unsafe actions in government workflows.
